Hands-on Tasks

Task 1: Rectangle Area

What to do:

  • Write a function area(length, width) that returns the area of a rectangle
  • Call it three times with different values and log each result
  • Bonus: Rewrite the same function as an arrow function with implicit return
function area(length, width) {
  return length * width;
}

console.log(area(5, 10));   // 50
console.log(area(7, 3));    // 21
console.log(area(12, 4));   // 48

// Bonus: Arrow function with implicit return
const areaArrow = (length, width) => length * width;

console.log(areaArrow(5, 10));  // 50
console.log(areaArrow(7, 3));   // 21
console.log(areaArrow(12, 4));  // 48

Task 2: Greeting with Default Parameters

What to do:

  • Write a function greetName(name) that returns "Hello, <name>!"
  • Use a default parameter: if no name is passed, it should default to "Guest"
  • Call it three times: with "Priya", with "Aarav", and with no argument
  • Bonus: Try greetName(null) � does it use the default? Why or why not?
function greetName(name = "Guest") {
  return `Hello, ${name}!`;
}

console.log(greetName("Priya"));   // Hello, Priya!
console.log(greetName("Aarav"));   // Hello, Aarav!
console.log(greetName());          // Hello, Guest!
console.log(greetName(null));      // Hello, null!

// Note: Default parameters only trigger for undefined, not for null!
// null is an explicit value, so the default doesn't apply.

Task 3: Temperature Converter

What to do:

  • Write an arrow function cToF(celsius) that converts Celsius to Fahrenheit
  • Formula: F = C * 9/5 + 32
  • Use implicit return
  • Test with: 0, 100, 37 (body temperature), and 45 (Jaipur summer)
const cToF = (celsius) => (celsius * 9/5) + 32;

console.log(cToF(0));    // 32 (freezing point)
console.log(cToF(100));  // 212 (boiling point)
console.log(cToF(37));   // 98.6 (body temperature)
console.log(cToF(45));   // 113 (Jaipur summer)

Bonus: Pure vs Impure Functions

What to do:

  • Write a PURE function double(n) that returns n * 2
  • Then write an IMPURE function addToTotal(n) that updates an outer variable let total = 0 and returns the new total
  • Call each function three times
  • Note how the impure one's output depends on previous calls
  • Write a comment: which one is easier to reason about, and why?
// PURE function - always returns same output for same input
function double(n) {
  return n * 2;
}

console.log(double(2));   // 4
console.log(double(5));   // 10
console.log(double(10));  // 20

// IMPURE function - output depends on outer state
let total = 0;

function addToTotal(n) {
  total += n;
  return total;
}

console.log(addToTotal(2));   // 2 (first call)
console.log(addToTotal(5));   // 7 (depends on previous call)
console.log(addToTotal(10));  // 17 (depends on all previous calls)

// The pure function (double) is easier to reason about because:
// - Same input always gives same output
// - No hidden dependencies on external state
// - Easier to test and debug
// - Can be used anywhere without side effects
