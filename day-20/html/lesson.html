Day 20 — Event loop & task queues

Lesson

Open DevTools → Console, or run node day-20/js/lesson.js (~12s — includes a 3 second busy-wait from the student doc). Source: ../js/lesson.js.

1 Single-threaded JS

One call stack, one synchronous statement at a time. Async work is delegated to the runtime; completions are queued and the event loop runs them when the stack is empty.

../js/lesson.js — Topic 1

2 Runtime architecture 
┌──────────────────────────────────┐
│ JS Engine (e.g. V8)              │
│   Stack          Heap            │
└──────────────────────────────────┘
          ▲ Event loop pulls when stack empty
          │
┌─────────────────────┐   ┌──────────────────┐
│ Microtask queue     │◀──│ Web APIs /       │
│ .then, queueMicro…  │   │ Node (timers,    │
└─────────────────────┘   │ I/O, fetch…)     │
┌─────────────────────┐   └──────────────────┘
│ Macrotask queue     │◀── timers, I/O, …
│ setTimeout, …       │
└─────────────────────┘

ASCII from Day 20 student doc

3 What the event loop does
  1. Wait for the call stack to be empty.
  2. Drain all microtasks.
  3. Run one macrotask.
  4. Repeat.

../js/lesson.js — Topic 3

4 setTimeout(0) mystery

Sync runs first, then all microtasks, then macrotasks. So order is 1, 4, 3, 2 — not 1, 2, 3, 4.

Follow-along: X, W (sync), then Y (micro), then Z (macro).

../js/lesson.js — Topic 4 (each demo isolated in its own Promise so logs stay clean)

5 Microtask vs macrotask

Microtasks: Promise.then / catch / finally, queueMicrotask. Macrotasks: setTimeout, I/O, UI events, requestAnimationFrame (browser).

Classic ordering exercise: A, F, C, E, D, B — see runnable script.

../js/lesson.js — Topic 5

6 Don't block the loop

Long synchronous loops keep the stack busy — setTimeout(0) still waits. The runnable lesson.js matches the doc with a 3 second busy-wait (see comments + blocking()).

../js/lesson.js — Topic 6

7 Practical implications
  • Promise callbacks run before the next timer.
  • setTimeout(0) means "after sync work + all microtasks", not "now".
  • Blocking the stack freezes UI / timers until you return.
  • await continuations are scheduled as microtasks (same phase as .then).
