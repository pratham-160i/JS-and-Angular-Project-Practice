Day 14 — Scope & closures

Lesson

Open DevTools → Console, or run node day-14/js/lesson.js. Source: ../js/lesson.js.

1 Lexical (static) scope

A function’s scope is fixed by where it is written in the source, not by where it is called. JavaScript uses lexical scope only.

const city = "Jaipur";
function outer() {
  const language = "Hindi";
  function inner() {
    const greeting = "Namaste";
    console.log(greeting, language, city);
  }
  inner();
}
outer();

2 The scope chain

Look up names in the current scope, then the next outer scope, until global. Inner scopes cannot see sibling inners’ bindings. Missing name → ReferenceError.

const x = 1;
function a() {
  const y = 2;
  function b() {
    console.log(x + y); // walks outward: 3
  }
  b();
}
a();

../js/lesson.js — Topic 2

3 What is a closure?

A function that remembers variables from the scope where it was created, even after that outer function has returned. Each call to a factory can create a separate closure with its own captured values.

function makeGreeter(name) {
  return function () {
    console.log(`Namaste, ${name}!`);
  };
}
const greetPriya = makeGreeter("Priya");
const greetAarav = makeGreeter("Aarav");
greetPriya();
greetAarav();

4 Practical closure patterns

Private state — counter, bank-style object with hidden balance. Memoization — cache results of an expensive function in a closed-over object.

function makeCounter() {
  let count = 0;
  return () => ++count;
}
function memoize(fn) {
  const cache = {};
  return (n) => (n in cache ? cache[n] : (cache[n] = fn(n)));
}

5 The var-in-loop bug

var is function-scoped: one shared i for the whole loop, so every callback sees the final value. let gives a fresh binding per iteration. Classic fixes: let, per-iteration IIFE, or bind / passing the value into the timer.

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // often 3, 3, 3
}
for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100); // 0, 1, 2
}

../js/lesson.js uses synchronous function arrays to show the same idea without waiting on timers.

6 IIFE — Immediately Invoked Function Expression

Runs once, creates a private scope. Common historical pattern: return a small API object while hiding let state in the closure.

(function () {
  const secret = "hidden";
})();
(function (city) {
  console.log(`Greetings from ${city}`);
})("Jaipur");
(() => { console.log(42); })();

const counterModule = (function () {
  let count = 0;
  return { inc: () => ++count, get: () => count };
})();
