Day 17 — ES6 classes

Lesson

Open DevTools → Console, or run node day-17/js/lesson.js. Source: ../js/lesson.js.

1 Class syntax basics

constructor runs on new User(…). Instance fields are own to each instance; methods live on User.prototype (shared).

class User {
  constructor(name, city) {
    this.name = name;
    this.city = city;
  }
  greet() {
    console.log(`Hi, I'm ${this.name} from ${this.city}`);
  }
}

Under the hood: typeof User === "function"; instances share methods via the prototype chain (Day 16).

2 Getters and setters

Getters/setters look like properties: read p.priceInRupees, assign p.priceInRupees = 100. Use _priceInPaise as a convention for internal state.

get priceInRupees() { return this._priceInPaise / 100; }
set priceInRupees(rupees) {
  if (rupees < 0) throw new Error("Price cannot be negative");
  this._priceInPaise = rupees * 100;
}

Follow-along: class T holds _c (Celsius); getter f returns _c * 9/5 + 32. For new T(100), t.f is 212.

../js/lesson.js — Topic 2 (+ follow-along)

3 extends and super

Subclass constructors must call super(…) before using this. Override methods; optionally call super.speak() from the child.

class Dog extends Animal {
  constructor(name, breed) {
    super(name);
    this.breed = breed;
  }
}

instanceof walks the prototype chain: a Dog is an Animal too.

../js/lesson.js — Topic 3

4 Static methods

Call static methods on the class (MathUtils.gst(1000)), not on instances. Factory pattern: User.fromEmail(email) returns new User(…).

static fromEmail(email) {
  const name = email.split("@")[0];
  return new User(name);
}

../js/lesson.js — Topic 4 (two blocks: MathUtils + User factory)

5 Private fields (#)

#balance is language-enforced: no property access from outside the class body.

class BankAccount {
  #balance;
  #transactions = [];
  // ...
}

Expose read-only views with getters (e.g. return a copy of #transactions from get history()).

../js/lesson.js — Topic 5

6 Custom Error subclasses

Subclass Error: call super(message), set this.name = this.constructor.name, add fields like code or field.

if (e instanceof ValidationError) {
  console.log(`[${e.code}] ${e.field}: ${e.message}`);
}

../js/lesson.js — Topic 6

7 Classes are sugar

ES6 class desugars to constructor functions + methods on .prototype — same model as Day 16, with clearer syntax.

// Pre-ES6: function Person + Person.prototype.greet
// ES6:    class Person { greet() { ... } }
