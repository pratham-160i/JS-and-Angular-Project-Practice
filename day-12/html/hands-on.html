Hands-on Tasks

Task 1: Safe JSON Parse

What to do: Write safeParse(str) that uses try/catch around JSON.parse(str). On success return the object; on failure log "Invalid JSON:" plus the error message and return null. Test with '{"name":"Priya"}' (valid) and '{"name":"Priya"' (broken). Expected return values and logs are noted as comments in hands-on.js.

function safeParse(str) {
  try {
    return JSON.parse(str);
  } catch (e) {
    console.log("Invalid JSON: " + e.message);
    return null;
  }
}

Task 2: Throw on Bad Age

What to do: Write setAge(age) that throws if typeof age !== "number" ("Age must be a number"), throws if age is not 0-120 ("Age must be 0-120"), else returns age. Wrap calls in try/catch. Test setAge(25), setAge("twenty"), setAge(200). Below: enter a number, or type twenty to simulate a non-number. Expected outcomes are comments in hands-on.js.

function setAge(age) {
  if (typeof age !== "number" || Number.isNaN(age)) {
    throw new Error("Age must be a number");
  }
  if (age < 0 || age > 120) {
    throw new Error("Age must be 0-120");
  }
  return age;
}

Task 3: Custom ValidationError on Email

What to do: Define class ValidationError extends Error with this.name = "ValidationError". Write validateEmail(email) that throws ValidationError if the email does not include @. Wrap in try/catch and branch on err instanceof ValidationError. Try priya@example.com and priya-no-at. Expected outcomes are comments in hands-on.js.

class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

function validateEmail(email) {
  if (!email.includes("@")) {
    throw new ValidationError("Email must include @");
  }
  return email;
}
