Day 15 — The this keyword

Lesson

Open DevTools → Console, or run node day-15/js/lesson.js. The script uses "use strict" so default this in plain calls is undefined (not window). Source: ../js/lesson.js.

1 What is this?

this is set when the function is called — it is the receiver of the call (who “asked” the function to run), not where the function was written.

function whoAmI() { console.log(this); }
whoAmI(); // strict: undefined
const user = { name: "Priya", whoAmI };
user.whoAmI(); // user object

2 The four binding rules

Default — plain fn(): undefined in strict / global in sloppy. Implicit — obj.method(): this is obj. Explicit — fn.call(x) / apply / bind: this is x. new — this is the new instance.

const obj = { n: 7, f() { return this.n; } };
const g = obj.f;
console.log(obj.f()); // 7
console.log(g());     // strict: throws (no receiver)

../js/lesson.js catches the loose-call error for Topic 2 follow-along.

3 call, apply, bind

.call(thisArg, a, b) — invoke now, args listed. .apply(thisArg, [a, b]) — invoke now, args array. .bind(thisArg, a) — returns a new function; can pre-fill arguments. A hard-bound function ignores later .call for this.

const greetPriya = greet.bind(u, "Jaipur");
greetPriya("English");
greetPriya.call({ name: "Aarav" }, "Tamil"); // still Priya + Jaipur

../js/lesson.js — Topic 3

4 Arrow functions — lexical this

Arrows have no own this; they close over the enclosing scope. Avoid arrows as object methods if you need this to be the object. Prefer them inside methods for callbacks (e.g. forEach) so this stays the outer receiver.

greetAll() {
  this.members.forEach((m) => {
    console.log(`Hi ${m}, team ${this.members.length}`);
  });
}

../js/lesson.js — Topic 4

5 this in classes

In constructor, this is the instance being built. In methods, this is whichever instance the method was called on — unless the method is passed around as a bare function (then you lose the receiver).

const greetFn = a.greet;
greetFn(); // TypeError — lost this

../js/lesson.js — Topic 5 (wrapped in try/catch)

6 Lost this — bind, wrapper, class field arrow

Passing c.inc as a callback drops the receiver. Fixes: c.inc.bind(c), () => c.inc(), or an instance arrow field inc = () => { ... } so the function itself closes over this.

setTimeout(c.inc.bind(c), 100);
setTimeout(() => c.inc(), 100);

class CounterArrow {
  count = 0;
  inc = () => { this.count++; };
}
