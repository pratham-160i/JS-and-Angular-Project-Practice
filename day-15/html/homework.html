Day 15 — The this keyword

Homework

Predict first where asked, then open the console. Reference: ../js/homework.js.

1 user.greet(), detached fn(), .call()

Build user = { name, greet } with greet logging this.name. Predict, then run: user.greet(), const fn = user.greet; fn(), user.greet.call({ name: "X" }).

const fn = user.greet;
fn(); // strict: TypeError — lost receiver
2 Callback bug — three fixes

Same issue as passing a class method to setTimeout without a receiver. Document: bind (permanent receiver), arrow wrapper (lexical c), class field arrow (method is an arrow, closes over this).

3 sum + .apply

Implement sum(...nums) with reduce. Call with sum.apply(null, [1,2,3,4,5]). Why apply? It expands an array into positional arguments—ideal when values already sit in an array.

4 Arrow + .bind

Top-level const f = () => console.log(this); f.bind({ x: 1 })(). Arrows have no own this; bind/call/apply do not change the lexical this they close over.
