Hands-on: In-class Tasks

Hands-on Tasks

Complete these tasks during class. Open the browser console (F12) and try each exercise.

Task 1: Hello, World

What to do:

  • Open Chrome DevTools → Console (F12 or right-click → Inspect → Console tab)
  • Type console.log("Hello, World!") and press Enter
  • Now declare a const for your name and log it
  • Then log "<your name> is learning JS" using string concatenation with +
console.log("Hello, World!");
const myName = "Pratham";
console.log(myName);
console.log(myName + " is learning JS");

Task 2: let vs const

What to do:

  • Create let age = 25. Log it.
  • Reassign age = 26. Log it again. Confirm the new value.
  • Create const PI = 3.14. Try to reassign PI = 3.15. Note the error.
  • Create let cityName (no value yet). Log it. Then assign "Jaipur". Log again.
let age = 25;
console.log(age);           // 25
age = 26;
console.log(age);           // 26

const PI = 3.14;
console.log(PI);            // 3.14
// PI = 3.15;              // Error: Assignment to constant variable

let cityName;
console.log(cityName);      // undefined
cityName = "Jaipur";
console.log(cityName);      // Jaipur

Task 3: Block Scope

What to do:

  • Create a block { let x = 10; console.log(x); }. Run it.
  • Now try to log x AFTER the block. Note the error.
  • Inside an if (true) { ... } block, declare let inside = "hi" and log it.
  • Try logging inside outside the if. Note the error.
{ 
    let x = 10; 
    console.log(x);          // 10
}
// console.log(x);          // Error: x is not defined

if (true) {
    let inside = "hi";
    console.log(inside);    // hi
}
// console.log(inside);    // Error: inside is not defined

Bonus: var Pitfall

What to observe:

  • Inside a for (var i = 0; i < 3; i++) { } loop, log i AFTER the loop ends.
  • Note that i is still accessible — it leaked out!
  • Now do the same with let i. Try logging i after — note the error.
  • In a comment, write WHY this matters.
// var leaks out of loop
for (var i = 0; i < 3; i++) {}
console.log(i);             // 3 (var leaks!)

// let is block-scoped
for (let j = 0; j < 3; j++) {}
// console.log(j);         // Error: j is not defined

// Why it matters: var can cause bugs because variables leak.
// let is safer and more predictable.
