Exercise 1: Declare All Primitive Types

What to do:

  • In a day2.js file, declare one variable of EACH primitive type
  • Log each value AND its typeof
const str = "Hello";
const num = 42;
const bigNum = 1000n;
const bool = true;
const sym = Symbol("unique");
let undefinedVal;
const nullVal = null;

console.log(str, typeof str);               // Hello string
console.log(num, typeof num);               // 42 number
console.log(bigNum, typeof bigNum);         // 1000n bigint
console.log(bool, typeof bool);             // true boolean
console.log(sym, typeof sym);               // Symbol(unique) symbol
console.log(undefinedVal, typeof undefinedVal);   // undefined undefined
console.log(nullVal, typeof nullVal);       // null object

Exercise 3: The 0.1 + 0.2 Problem

What to do:

  • Compute 0.1 + 0.2. Log the result
  • Use the multiply-to-paise trick: compute (10 + 20) / 100 and confirm you get exactly 0.3
// The JavaScript surprise!
let result = 0.1 + 0.2;
console.log(result);                        // 0.30000000000000004 (surprise!)
console.log(result === 0.3);                // false

// The solution: work with integers (paise)
let paise1 = 10;   // 0.10 rupees
let paise2 = 20;   // 0.20 rupees
let total = (paise1 + paise2) / 100;
console.log(total);                         // 0.3 (exactly!)
console.log(total === 0.3);                 // true

// Why? Floating-point numbers in binary can't represent 0.1 and 0.2 exactly.

Exercise 4: User Card with Multi-line Template Literal

What to do:

  • Build a "user card" string using a multi-line template literal
  • Include name, age, city, and one fact about you
  • In a comment, explain when YOU should use null vs let it be undefined
const name = "Rahul";
const age = 24;
const city = "Delhi";
const fact = "I love coding!";

const userCard = `
========== USER CARD ==========
Name: ${name}
Age: ${age}
City: ${city}
Fun Fact: ${fact}
================================
`;

console.log(userCard);

// When to use null vs undefined:
// undefined: "I didn't set a value" (automatic, like uninitialized variables)
// null: "I explicitly set this to empty" (intentional choice by programmer)
// Use null when you want to say "this value is intentionally empty"
// Let it be undefined when a value hasn't been assigned yet.

Bonus: Number Quirks

What to observe:

  • Compute 0.1 + 0.2. Log the result and compare to 0.3 with ===. Note the surprise.
  • Try Number.isInteger(5), Number.isInteger(5.0), Number.isInteger(5.5)
  • Try 1 / 0 and "abc" * 2. Note the special values
// 0.1 + 0.2 surprise
console.log(0.1 + 0.2);                     // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3);             // false!

// Number.isInteger checks
console.log(Number.isInteger(5));           // true
console.log(Number.isInteger(5.0));         // true (5.0 is same as 5)
console.log(Number.isInteger(5.5));         // false

// Special values
console.log(1 / 0);                         // Infinity
console.log("abc" * 2);                     // NaN (Not a Number)
console.log(typeof Infinity);               // "number"
console.log(typeof NaN);                    // "number"

// Cool fact: NaN is never equal to itself!
console.log(NaN === NaN);                   // false
console.log(Number.isNaN(NaN));             // true (correct way to check)
