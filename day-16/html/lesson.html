Day 16 — Prototypes & prototype chain

Lesson

Open DevTools → Console, or run node day-16/js/lesson.js. Source: ../js/lesson.js.

1 [[Prototype]] and Object.prototype

Every object has an internal link to a prototype object. Property reads walk the chain until Object.prototype, then null.

const user = { name: "Priya" };
Object.getPrototypeOf(user) === Object.prototype; // true
user.toString(); // from Object.prototype

../js/lesson.js — Topic 1

2 Object.create

Creates an object with a chosen prototype. hasOwnProperty distinguishes own keys from inherited ones.

const a = { x: 1 };
const b = Object.create(a);
b.y = 2;
// b.x from prototype; b.y own

../js/lesson.js — Topic 2 (animal / dog + follow-along)

3 Prototype chain — read vs write

Reading walks up the chain. Assigning creates an own property on the target object and may shadow an inherited name without changing the prototype.

child.firstName = "Anaya"; // own on child
// parent.firstName unchanged

../js/lesson.js — Topic 3

4 hasOwnProperty vs in vs Object.hasOwn

hasOwnProperty / Object.hasOwn: only own keys. in: anywhere on the prototype chain (including Object.prototype).

"toString" in dog; // true
dog.hasOwnProperty("toString"); // false

../js/lesson.js — Topic 4

5 Constructor functions & .prototype

Instances link to Constructor.prototype. Shared methods go on that object so every instance reuses the same function. Subclassing: Sub.prototype = Object.create(Super.prototype) and fix constructor.

User.prototype.greet = function () { ... };
Admin.prototype = Object.create(User.prototype);
Admin.prototype.constructor = Admin;

../js/lesson.js — Topic 5

6 Do not extend built-in prototypes

Avoid Array.prototype.last = … in real code: global side effects, future spec collisions, odd for...in behaviour. Prefer a small utility function.

The runnable lesson does not patch Array.prototype; it only logs the rationale and uses last(arr).
