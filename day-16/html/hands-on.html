Day 16 — Prototypes & prototype chain

Hands-on

Open DevTools → Console, or run node day-16/js/hands-on.js. Work through each task in ../js/hands-on.js.

Task 1 Inspect a prototype
  1. Create const arr = [1, 2, 3];
  2. Log Object.getPrototypeOf(arr). What object is it?
  3. Log Object.getPrototypeOf(Object.getPrototypeOf(arr)). What is it?
  4. Log one more Object.getPrototypeOf(…) on that result. What value ends the chain?
  5. In a comment in the JS file, draw the full chain from arr to the end.

Answers (run the script): first link is Array.prototype, then Object.prototype, then null.

Task 2 Build with Object.create
  1. Create vehicle with method start() that logs this.name + " starting" (or a template string).
  2. Use Object.create(vehicle) for car; set car.name = "Tata Nexon".
  3. Make bike the same way with name = "Royal Enfield".
  4. Call start() on both.
  5. Use hasOwnProperty and in to confirm name is own and start is inherited.

See Task 2 in ../js/hands-on.js.

Task 3 Constructor function inheritance
  1. Person(name) — set this.name = name.
  2. Person.prototype.greet = function () { console.log("Hi, I'm " + this.name); }
  3. Student(name, school) — Person.call(this, name) and this.school = school.
  4. Link Student.prototype to inherit from Person.prototype using Object.create(Person.prototype). Restore Student.prototype.constructor = Student.
  5. Add Student.prototype.study logging this.name + " studies at " + this.school.
  6. new Student("Riya", "IIT Delhi") — call greet and study.

See Task 3 in ../js/hands-on.js.

Bonus hasOwnProperty vs in

Create const dog = Object.create({ species: "Canis" }); dog.name = "Bruno";

Predict, then run and check:

  • dog.hasOwnProperty("name")
  • dog.hasOwnProperty("species")
  • "name" in dog
  • "species" in dog
  • "toString" in dog

In a comment in the JS file, write the one-line rule for when to use which.

Bonus block at the bottom of ../js/hands-on.js.
