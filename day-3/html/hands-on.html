Hands-on Tasks

Task 1: GST Calculator

What to do:

  • Take const price = 850. Compute the GST at 18%.
  • Compute the total (price + GST)
  • Use a template literal to log: "Price: ₹850, GST: ₹153, Total: ₹1003"
  • Try with price = 1234.50. Note any floating-point oddness.
const price = 850;
const gstRate = 18;
const gst = (price * gstRate) / 100;
const total = price + gst;

console.log(`Price: ₹${price}, GST: ₹${gst}, Total: ₹${total}`);
// Price: ₹850, GST: ₹153, Total: ₹1003

// Now try with decimal
const price2 = 1234.50;
const gst2 = (price2 * gstRate) / 100;
const total2 = price2 + gst2;

console.log(`Price: ₹${price2}, GST: ₹${gst2}, Total: ₹${total2}`);
// Watch for floating-point quirks!

Task 2: == vs ===

What to do:

  • Compare each pair with BOTH == and ===
  • Predict each result before running
  • For each surprising one, write a comment explaining the coercion
// Pair 1: 5 and "5"
console.log(5 == "5");                      // true (coercion: string converts to number)
console.log(5 === "5");                     // false (different types)

// Pair 2: 0 and false
console.log(0 == false);                    // true (coercion: false converts to 0)
console.log(0 === false);                   // false (different types)

// Pair 3: null and undefined
console.log(null == undefined);             // true (special case in JS!)
console.log(null === undefined);            // false (different types)

// Pair 4: "" and false
console.log("" == false);                   // true (coercion: empty string converts to 0, false converts to 0)
console.log("" === false);                  // false (different types)

// Pair 5: "abc" and "abc"
console.log("abc" == "abc");                // true (same value)
console.log("abc" === "abc");               // true (same value AND same type)

// Key lesson: Always use === to avoid unexpected type coercion!

Task 3: Logical Operators

What to do:

  • Take const age = 22, const hasEmail = true
  • Build canRegister using && — true if age ≥ 18 AND hasEmail
  • Take const userName = "" (empty string). Use || to default it to "Guest"
  • Use ! to flip a boolean
  • Log all three results
const age = 22;
const hasEmail = true;
const canRegister = age >= 18 && hasEmail;
console.log("Can Register:", canRegister);  // true

const userName = "";
const displayName = userName || "Guest";
console.log("Display Name:", displayName);  // Guest

const isLoggedIn = false;
const isLoggedOut = !isLoggedIn;
console.log("Is Logged Out:", isLoggedOut); // true

Bonus: Ternary Practice

What to do:

  • Use a ternary to set const fee = age >= 65 ? 0 : 100. Try with age = 70 and age = 30
  • Use a ternary inside a template literal: `Discount: ${age >= 65 ? "Yes" : "No"}`
  • Write a ternary that returns "high", "medium", or "low" based on a score variable (nested ternary)
// Ternary for senior citizen discount
let age1 = 70;
const fee1 = age1 >= 65 ? 0 : 100;
console.log(`Age ${age1}: Fee = ₹${fee1}`);   // Age 70: Fee = ₹0

let age2 = 30;
const fee2 = age2 >= 65 ? 0 : 100;
console.log(`Age ${age2}: Fee = ₹${fee2}`);   // Age 30: Fee = ₹100

// Ternary in template literal
console.log(`Age ${age1}: Discount ${age1 >= 65 ? "Yes" : "No"}`);

// Nested ternary for score rating (keep readable!)
const score = 75;
const rating = score >= 80 ? "high" : score >= 60 ? "medium" : "low";
console.log(`Score ${score}: Rating = ${rating}`);  // Score 75: Rating = medium
