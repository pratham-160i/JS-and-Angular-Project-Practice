Day 18 — Promises & async flow

Homework

The runnable file is ../js/homework.js (also: node day-18/js/homework.js). Below each problem is the reference solution — same logic as that file.

1 wait(ms)

Implement wait(ms) returning a Promise that resolves after ms milliseconds. Then chain:

wait(500).then(() => wait(500)).then(() => console.log("1s"));

Reference solution

function wait(ms) {
  return new Promise((resolve) => {
    setTimeout(resolve, ms);
  });
}

wait(500)
  .then(() => wait(500))
  .then(() => console.log("1s"));

In homework.js this runs inside task1().

2 fetchData(id) + chain

fetchData(id) should randomly resolve or reject (~50/50). Chain three sequential .then calls and end with .catch. Run multiple times to see both full success and early failure.

Reference solution

function fetchData(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const success = Math.random() >= 0.5;
      if (success) resolve(`Data for ${id}`);
      else reject(new Error(`Failed for ${id}`));
    }, 120);
  });
}

fetchData(1)
  .then((data) => {
    console.log(data);
    return fetchData(2);
  })
  .then((data) => {
    console.log(data);
    return fetchData(3);
  })
  .then((data) => {
    console.log(data);
  })
  .catch((err) => {
    console.error(err.message);
  });

The file also loops this chain several times in runHomework() so you see different outcomes.

3 Promise.race timeout

Use Promise.race with [fetchUser(7), wait(2000).then(() => Promise.reject(new Error("timeout")))] so a request that takes longer than 2 seconds loses to the timeout.

The script also runs a fast fetchUser so you see the user win the race when the network is quicker than 2s.

Reference solution

function fetchUser(id) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id, name: "Priya" }), 500); // fast path
  });
}

Promise.race([
  fetchUser(7),
  wait(2000).then(() => Promise.reject(new Error("timeout"))),
])
  .then((user) => console.log(user))
  .catch((err) => console.error(err.message));

// Slow fetch (>2s) — timeout wins:
function fetchUserSlow(id) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id, name: "Priya" }), 3500);
  });
}

Promise.race([
  fetchUserSlow(7),
  wait(2000).then(() => Promise.reject(new Error("timeout"))),
])
  .then((user) => console.log(user))
  .catch((err) => console.error(err.message)); // "timeout"

In homework.js: task3FastFetchWins and task3TimeoutWins.

4 Promise.any flaky fetches

Promise.any returns the first fulfilled promise and ignores rejections until every input rejects (then you get an AggregateError).

Includes a deterministic demo (only one API succeeds) plus two batches of random flaky fetches.

Reference solution

function flakyFetch(name) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (Math.random() >= 0.5) resolve(`${name} success`);
      else reject(new Error(`${name} failed`));
    }, 80);
  });
}

Promise.any([
  flakyFetch("API-1"),
  flakyFetch("API-2"),
  flakyFetch("API-3"),
])
  .then((result) => {
    console.log("First success:", result);
  })
  .catch((err) => {
    console.log("All failed", err.name);
  });

In homework.js: flakyFetch, flakyFixed, task4Deterministic, task4RandomBatch.
