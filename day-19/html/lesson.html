Day 19 — async / await

Lesson

Open DevTools → Console, or run node day-19/js/lesson.js (~12s). Source: ../js/lesson.js.

1 async always returns a Promise

Even return "Namaste" inside async function greet() becomes a fulfilled Promise. Callers use .then or await to read the value.

Same idea without sugar: function greetOld() { return Promise.resolve("Namaste"); }

../js/lesson.js — Topic 1

2 await pauses this function

await only inside async functions (or top-level in ES modules). It pauses that function until the Promise settles; other code keeps running.

Follow-along: console.log(f()) prints a Promise; await f() prints the resolved value 1.

../js/lesson.js — Topic 2

3 try / catch / finally

Use the same error model as sync code: try { await fetchUser(id) } catch (err) { … } finally { … }.

../js/lesson.js — Topic 3 (showUser)

4 Sequential vs parallel

Three independent awaits in a row run sequentially (slow). await Promise.all([...]) starts them together (fast).

../js/lesson.js — Topic 4 (fetchProduct with ~350ms each)

5 The forEach trap

forEach ignores the Promise your async callback returns — the loop finishes immediately. Use for...of for sequential work or Promise.all(ids.map(...)) for parallel.

../js/lesson.js — Topic 5

6 Top-level await

Allowed in ES modules. In classic scripts, use (async () => { … })();. The runnable lesson uses a mock config fetch (no real fetch("/api/config")).

../js/lesson.js — Topic 6

7 Mixing with .then

An async function returns a Promise; callers may use getUserName(7).then(...) or await getUserName(7) inside another async function.

../js/lesson.js — Topic 7
