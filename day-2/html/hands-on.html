Day 2: Variables

Hands-on: In-class Tasks

Hands-on Tasks

Task 1: typeof Exploration

What to do:

  • In the console, log typeof for each: 42, "hello", true, undefined, null, 100n, Symbol("x")
  • Note which one returns a SURPRISING type
  • In a comment, write down which one and why
console.log(typeof 42);                 // "number"
console.log(typeof "hello");            // "string"
console.log(typeof true);               // "boolean"
console.log(typeof undefined);          // "undefined"
console.log(typeof null);               // "object" -- SURPRISE! Null is NOT an object!
console.log(typeof 100n);               // "bigint"
console.log(typeof Symbol("x"));        // "symbol"

// Interesting fact: null returns "object" due to a quirk in JavaScript!
// This is a historical bug that couldn't be fixed without breaking code.

Task 2: Template Literals

What to do:

  • Create variables: const name = "Aarav", const age = 22, const city = "Jaipur"
  • Build the string "Aarav, 22, from Jaipur" using OLD-style concatenation with +
  • Build the same string using a TEMPLATE LITERAL
  • Then build a 3-line "card" using a multi-line template literal
const name = "Aarav";
const age = 22;
const city = "Jaipur";

// OLD-style concatenation with +
let oldStyle = name + ", " + age + ", from " + city;
console.log(oldStyle);                  // Aarav, 22, from Jaipur

// NEW-style with template literal
let newStyle = `${name}, ${age}, from ${city}`;
console.log(newStyle);                  // Aarav, 22, from Jaipur

// Multi-line template literal (3-line card)
let card = `Name: ${name}
Age: ${age}
City: ${city}`;
console.log(card);

Task 3: null vs undefined

What to do:

  • Declare let user; (no value). Log it. Log its typeof
  • Now set user = null. Log it. Log its typeof
  • Compare: null == undefined and null === undefined. Note the difference.
let user;
console.log(user);                      // undefined
console.log(typeof user);               // "undefined"

user = null;
console.log(user);                      // null
console.log(typeof user);               // "object" (quirk!)

// Loose equality (==) treats them as equal
console.log(null == undefined);         // true

// Strict equality (===) treats them as different
console.log(null === undefined);        // false

// Key difference:
// undefined = "no value was set"
// null = "I intentionally set this to no value"
