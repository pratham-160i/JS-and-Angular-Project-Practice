Day 21 — Clones, immutability & modern objects

Lesson

Open DevTools → Console, or run node day-21/js/lesson.js. Source: ../js/lesson.js.

1 Shallow vs deep clone

Shallow: { ...original } and Object.assign({}, original) copy top-level keys only — nested objects stay shared. JSON deep-clones plain data but loses Date, Map, undefined, functions.

const original = {
  name: "Priya",
  address: { city: "Jaipur", pin: 302001 },
  hobbies: ["reading", "trekking"],
};
const a = { ...original };
const b = Object.assign({}, original);
const c = JSON.parse(JSON.stringify(original));
a.address.city = "Mumbai"; // original.address changes too
const tricky = { d: new Date(), m: new Map(), u: undefined };
JSON.parse(JSON.stringify(tricky)); // Date → string, Map → {}, u dropped

2 structuredClone

Built-in deep clone for many built-ins. Cannot clone functions / DOM / some class prototypes (DataCloneError).

const copy = structuredClone(original);
copy.nested.city = "Mumbai";
// original.nested unchanged
// copy.date instanceof Date → true
// structuredClone({ fn: () => {} }); // throws

const o = { x: { y: 1 } };
const a = { ...o };
a.x.y = 99;        // o.x.y becomes 99 (shared nested)
const b = structuredClone(o);
b.x.y = 50;        // o.x.y unchanged by b alone (doc: check after shallow step)

3 Object.freeze & deep freeze

Object.freeze is shallow — nested objects stay mutable unless you recurse (deepFreeze).

const config = Object.freeze({ host: "api.example.com", port: 8080 });
config.port = 9000; // ignored (strict: TypeError)

const user = Object.freeze({ name: "Priya", address: { city: "Jaipur" } });
user.address.city = "Mumbai"; // still works — nested not frozen

function deepFreeze(obj) {
  Object.values(obj).forEach((v) => {
    if (v && typeof v === "object" && !Object.isFrozen(v)) deepFreeze(v);
  });
  return Object.freeze(obj);
}

4 Immutable updates

Spread and replace nested slices — never mutate inputs in place when you need a predictable previous state.

const u1 = { ...user, age: 26 };
const u2 = { ...user, address: { ...user.address, city: "Mumbai" } };
const u3 = { ...user, hobbies: [...user.hobbies, "swimming"] };
const u4 = { ...user, hobbies: user.hobbies.filter((h) => h !== "trekking") };
const t1 = tasks.map((t) => (t.id === 1 ? { ...t, done: true } : t));

5 Advanced destructuring

Defaults, renames, nested picks, array rest, and parameter patterns (common in React props).

const { name, role = "user" } = { name: "Priya" };
const { name: userName, role: userRole = "user" } = { name: "Aarav" };
const { api: { host: apiHost, port: apiPort } } = config;
const [first = "?", second = "?", ...rest] = [1, 2, 3, 4, 5];

6 Computed property keys

Use [expression] as a key — handy for generic updateField(obj, key, value).

const field = "city";
const b = { [field]: "Mumbai" };

function updateField(obj, key, value) {
  return { ...obj, [key]: value };
}

7 Optional chaining (?.) & nullish (??)

?. short-circuits on null/undefined. ?? falls back only for null or undefined (not 0 or "").

const city = response?.user?.profile?.city;
const cityOrDefault = response?.user?.profile?.city ?? "Unknown";
const upper = response?.user?.name?.toUpperCase?.();
const first = response?.user?.tags?.[0] ?? "no tags";
